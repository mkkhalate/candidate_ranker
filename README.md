# Redrob Candidate Ranking Engine

A CPU‑only offline pipeline that ranks 100K+ candidates against a job description (JD) using hybrid retrieval, cross‑encoder reranking, and deterministic JD‑aligned scoring. Built for the **Intelligent Candidate Discovery & Ranking Challenge**.

## Overview

The pipeline ingests a job description (`job_description.md`) and a JSONL file of candidate profiles (`candidates.jsonl` or `.gz`), and outputs a ranked list of the top‑100 candidates with a **composite score** and **human‑readable reasoning** for each.

It is designed to be **CPU‑only** and runs entirely offline after a one‑time pre‑computation step.

### Pipeline Steps

1. **Preprocessing** – Builds a text blob and extracts 35+ structural features per candidate.
2. **Hybrid Retrieval** – FAISS (dense) + BM25 (sparse) reduces 100K → top‑500.
3. **Cross‑Encoder Reranking** – Deep semantic scoring reduces 500 → top‑200.
4. **Deterministic Scoring** – JD‑aligned weighted formula with explicit penalties (top‑200 → top‑100).
5. **Explainability** – Generates a reasoning string for each candidate in the final top‑100.

---

## Key Features

- **Strict JD Alignment** – Enforces 5–9 years experience, penalises title‑chasers, framework‑only profiles, CV/speech primary without NLP/IR, consulting‑only careers, non‑India location, and candidates with weak shipping evidence.
- **Rich Feature Set** – Over 35 features including semantic match, skill depth, tenure stability, production evidence, external validation, evaluation framework experience, and JD‑specific flags.
- **Deterministic & Transparent** – Scoring is rule‑based, not a black‑box model, making it easy to debug and adjust.
- **Full Explainability** – Every candidate in the final output includes a clear reasoning string (e.g., *“Senior ML Engineer at Flipkart. semantic match: 1.00. strong skills in … shipped a ranking/search system.”*).
- **CPU‑Only** – All models (BGE, Cross‑Encoder, FAISS, BM25, XGBoost) run on CPU; no GPU required.

## Installation

### 1. Clone the repository

```bash
git clone https://github.com/mkkhalate/candidate_ranker.git
```



## Pre‑computation

Before running the ranking pipeline, you must generate the required model artefacts. This step is **run once** after you have placed your input files in `./input/`.

### 🏃 Speed Path (MiniLM)

**Linux / macOS (Bash):**
```bash
export OMP_NUM_THREADS=$(nproc) && export MKL_NUM_THREADS=$(nproc) && python precompute.py --input_dir ./input --models_dir ./models --embed_model_name "all-MiniLM-L6-v2" --embed_batch_size 128
```

**Windows (CMD):**
```cmd
set OMP_NUM_THREADS=%NUMBER_OF_PROCESSORS%
set MKL_NUM_THREADS=%NUMBER_OF_PROCESSORS%
python precompute.py --input_dir ./input --models_dir ./models --embed_model_name "all-MiniLM-L6-v2" --embed_batch_size 128
```

**Windows (PowerShell):**
```powershell
$env:OMP_NUM_THREADS=$env:NUMBER_OF_PROCESSORS
$env:MKL_NUM_THREADS=$env:NUMBER_OF_PROCESSORS
python precompute.py --input_dir ./input --models_dir ./models --embed_model_name "all-MiniLM-L6-v2" --embed_batch_size 128
```

Why %NUMBER_OF_PROCESSORS% / $env:NUMBER_OF_PROCESSORS?
It automatically detects your total logical CPU cores (e.g., 8, 12, 16) – no need to hardcode 8.


**What it does:**

- Streams `candidates.jsonl.gz` and builds text blobs.
- Generates BGE embeddings (768‑d) for each candidate.
- Builds and saves the FAISS index (`faiss_index.bin`).
- Builds and saves the BM25 index (`bm25_index.pkl`).
- Downloads and saves the embedding model and Cross‑Encoder locally (offline use).
- Trains an XGBoost ranker (using a synthetic rubric) and saves it (optional).

**Options:**

| Argument | Default | Description |
|----------|---------|-------------|
| `--input_dir` | `./input` | Path to input folder |
| `--models_dir` | `./models` | Path to store models |
| `--embed_batch_size` | `128` | Batch size for embedding generation |
| `--embed_model_name` | `BAAI/bge-base-en-v1.5` | Embedding model name |
| `--cross_encoder_name` | `cross-encoder/ms-marco-MiniLM-L-6-v2` | Cross‑Encoder model name |
| `--max_candidates` | `0` (all) | Limit number of candidates for testing |

After pre‑computation completes, you are ready to run the ranking pipeline.

## Running the Pipeline

```bash
python src/main.py --input_dir ./input --output_dir ./output --models_dir ./models
```

**Options:**

| Argument | Default | Description |
|----------|---------|-------------|
| `--input_dir` | `./input` | Input folder with JD and candidates |
| `--output_dir` | `./output` | Output folder for results |
| `--models_dir` | `./models` | Path to pre‑computed models |
| `--top_k` | `100` | Number of final candidates to output |
| `--retrieval_k` | `500` | Candidates retrieved by FAISS+BM25 |
| `--rerank_k` | `200` | Candidates passed to cross‑encoder |
| `--batch_size` | `32` | Cross‑encoder batch size |
| `--verbose_preprocess` | `False` | Log every candidate during preprocessing |

The pipeline will print progress and timings. At the end, you will find:

- `submission.csv` – the final ranked candidates.
- `ranking_log.json` – detailed information per candidate.
- `execution_timings.json` – step‑by‑step performance metrics.

## Scoring Logic

The final score is a **deterministic composite** of five components:

1. **Semantic Match (25%)** – Weighted combination of cross‑encoder score (70%) and BM25 rough score (30%).
2. **Skill Depth (20%)** – Based on expert/advanced skill counts, total skill score, and verified assessments.
3. **Experience (15%)** – Evaluates years of experience (with a boost for 5‑9 years), tenure average, stability, and seniority.
4. **Proof of Shipping (20%)** – Emphasises ranking/search system shipping evidence, production deployment, product‑company experience, external validation, and evaluation framework experience.
5. **Logistics (20%)** – Location match (India + preferred cities), notice period (≤30 days ideal), and open‑to‑work signal.

**Explicit Penalties** (subtracted from the composite):

- **Title‑chaser** (`avg_tenure < 1.5y` and ≥3 jobs): –0.30
- **Framework enthusiast** (only LangChain‑style frameworks, no underlying ML depth): –0.35
- **CV/speech primary without NLP/IR** (>40% senior skills in CV/speech and <2 NLP skills): –0.50
- **No shipped ranking system** and **ranking evidence < 0.3**: –0.20
- **Non‑India based**: –0.15
- **Consulting‑only** (no product‑company experience): –0.60 (pardoned to –0.20 if product experience exists)

**Integrity Penalties** (small deductions for honeypot flags, zero‑usage expert skills, company age anomalies, etc.)

These penalties ensure the ranking strictly follows the JD’s explicit “Things we explicitly do NOT want” section.

## Customisation

### Adjusting Weights / Penalties

Edit the `_recruiter_fit_score` function in `src/scorer.py`. The weights are clearly commented.

### Adding New Features

1. Add extraction logic in `src/preprocessor.py` inside `extract_features()`.
2. Include the new feature in the `features` dictionary returned.
3. In `scorer.py`, add the feature to the appropriate component (e.g., `proof_quality`) and update the reasoning labels in `explainer.py`.

### Changing the Experience Filter

The experience filter is applied in `run_deterministic_recruiter_scoring()` in `scorer.py`. Modify the condition `if 5.0 <= exp <= 9.0` to suit your JD.
